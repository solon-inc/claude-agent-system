# エージェントシステムを使った企画から開発までのユーザージャーニー

## 概要
TaskManager、Developer、DataGathererの3つのエージェントを活用して、アイデアから製品リリースまでを効率的に進めるユーザージャーニーです。

---

## 🎯 Phase 1: アイデア・企画フェーズ（Day 1-7）

### 1.1 市場調査・情報収集
**使用エージェント**: DataGatherer

```bash
# DataGathererを起動
claude code --agent datagatherer

# コマンド例
"ECサイトの最新トレンドを調査して"
"Shopifyの競合分析をして"
"2025年のEコマース市場動向を調べて"
```

**成果物**:
- knowledge_base/EXTERNAL_INSIGHTS.md に市場調査結果
- reports/weekly/に競合分析レポート
- Slackに重要な発見を自動通知

### 1.2 チーム情報の整理
**使用エージェント**: DataGatherer

```bash
# 初回セットアップ
"プロジェクトを初期化"

# インタビュー形式で情報収集
Q1: プロジェクト名を教えてください
→ "NextGen ECプラットフォーム"

Q2: チームメンバーと役割を教えてください
→ "山田太郎（PM）、鈴木花子（フロントエンド）、田中次郎（バックエンド）"
```

## 📋 Phase 2: 計画・要件定義フェーズ（Day 8-14）

### 2.1 Linearプロジェクト作成
**使用エージェント**: TaskManager

```bash
# TaskManagerを起動
claude code --agent taskmanager

# プロジェクト作成
"新規プロジェクトを開始"
→ タイプ: business
→ プロジェクト名: "NextGen EC - 事業企画"
```

**自動生成されるマイルストーン**:
- [Milestone] 市場調査完了
- [Milestone] ビジネスモデル確定
- [Milestone] 事業計画書承認

### 2.2 要件の詳細化
**使用エージェント**: TaskManager + DataGatherer

```bash
# TaskManager
"ユーザーストーリーを作成"
→ "商品検索機能の実装"
→ "決済システムの統合"
→ "在庫管理システムの構築"

# DataGatherer
"決済システムのベストプラクティスを調査して"
"StripeとPayPalの比較分析をして"
```

### 2.3 会議・意思決定の記録
**使用エージェント**: DataGatherer

```bash
# 要件定義会議の記録
"要件定義会議の議事録を作成"

# 会議音声の文字起こし
"inboxフォルダの会議音声を処理して"
→ 自動的に決定事項とアクションアイテムを抽出
```

## 🚀 Phase 3: 開発フェーズ（Day 15-60）

### 3.1 開発タスクの実装
**使用エージェント**: Developer

```bash
# Developerを起動
claude code --agent developer

# Linearタスクを受け取って実装
"Linear issue DEV-123を実装"
→ 自動的にブランチ作成: feature/DEV-123-product-search
→ 実装開始
→ テスト作成
→ Linear上で進捗更新
```

### 3.2 日々の開発サイクル
**典型的な1日の流れ**:

**朝（9:00）**:
```bash
# DataGatherer
"今日の状況をまとめて"
→ 昨日の進捗、本日のタスク、ブロッカーを集約

# TaskManager
"アクティブなタスクを確認"
→ 優先順位順にタスク一覧表示
```

**日中（10:00-17:00）**:
```bash
# Developer
"DEV-124の実装を開始"
→ コード実装
→ 単体テスト作成
→ Linearステータス更新

# 技術的な課題発生時
"このエラーを調査"
→ DataGathererが自動起動
→ 解決策を調査してDeveloperに引き継ぎ
```

**夕方（18:00）**:
```bash
# Developer
"PRを作成"
→ 自動的にLinearと連携
→ レビュー依頼

# DataGatherer
"日次レポートを作成"
→ 完了タスク、進行中タスク、明日の予定
```

### 3.3 スプリント管理
**使用エージェント**: TaskManager

```bash
# 2週間スプリント
"新しいスプリントを開始"
"スプリントバックログを作成"
"バーンダウンチャートを確認"
```

## 🎉 Phase 5: リリース・運用フェーズ（Day 76+）

### 5.1 リリース準備
**使用エージェント**: TaskManager + Developer

```bash
# TaskManager
"リリースチェックリストを作成"
"デプロイ手順を文書化"

# Developer
"本番環境へのデプロイ準備"
"リリースブランチを作成"
```

### 5.2 継続的な改善
**使用エージェント**: 全エージェント連携

```bash
# 毎週月曜日
DataGatherer: "週次レポートを作成"
TaskManager: "今週のスプリント計画を立てて"

# 毎日
DataGatherer: "プロジェクト情報を収集して"
→ Linear、GitHub、Slackから自動収集
→ 重要な変更を検知して通知

# 月次
TaskManager: "プロジェクトの振り返りを実施"
DataGatherer: "月次分析レポートを作成"
```

## 📊 エージェント連携パターン

### パターン1: 調査→計画→実装
```
DataGatherer（調査）
    ↓
TaskManager（タスク化）
    ↓
Developer（実装）
```

### パターン2: 問題解決フロー
```
Developer（問題発見）
    ↓
DataGatherer（解決策調査）
    ↓
TaskManager（対応タスク作成）
    ↓
Developer（修正実装）
```

### パターン3: 定期レビュー
```
DataGatherer（情報収集）
    ↓
TaskManager（分析・計画）
    ↓
全エージェント（実行）
```

## 💡 成功のポイント

### 1. 各エージェントの強みを活かす
- **DataGatherer**: 情報収集と分析に特化
- **TaskManager**: 計画とプロジェクト管理に特化
- **Developer**: 実装と技術的問題解決に特化

### 2. 自動化を最大限活用
- 定期的な情報収集は自動実行
- Linearステータスは自動更新
- Slack通知で重要情報を即座に共有

### 3. 情報の一元管理
- すべての情報はLinearに集約
- ドキュメントはGitHubで自動管理
- 会議記録は構造化して保存

### 4. コミュニケーションの効率化
- エージェント間の引き継ぎは自動
- 重要な決定はすぐに記録
- 進捗は常に可視化

1. **開発速度の向上**: 30-50%の効率化
2. **情報の透明性**: 全メンバーが最新情報にアクセス可能
3. **品質の向上**: 自動テストとレビューによる品質保証
4. **ナレッジの蓄積**: すべての決定と学びが記録される
